# Clyde demo recordings

The animated screenshots in the top-level [`README.md`](../README.md) are
generated by [VHS](https://github.com/charmbracelet/vhs) from the `.tape`
scripts in this directory.

| Tape | Output | Shows |
|------|--------|-------|
| [`hero.tape`](hero.tape) | `hero.gif` | Boot splash тЖТ full dashboard (no keys pressed) |
| [`observability.tape`](observability.tape) | `observability.gif` | `a`/`u`/`b`/`c` panel jumps, active-mode scrolling, `тМГn` hook notification answered with `y` |
| [`workspace.tape`](workspace.tape) | `workspace.gif` | `e` explorer тЖТ `/` fuzzy search тЖТ viewer тЖТ `f` fullscreen тЖТ `/`+`n` find тЖТ `i` insert тЖТ `:q!` |
| [`customization.tape`](customization.tape) | `customization.gif` | `?` settings overlay, live theme cycling, mascot persona swap |

Every keypress in these tapes is a real binding taken from
`internal/adapters/tui/keys.go` тАФ if a tape stops matching the app,
that's a bug in one of the two.

## Why these are reproducible

Every tape boots `clyde --demo`, which runs the TUI on a deterministic
in-memory fixture тАФ no live Claude Code session reads, no hook server, no
network. Each tape also exports throwaway `XDG_CONFIG_HOME` /
`XDG_CACHE_HOME` dirs before launching, because closing the settings
overlay (and `remember_layout`) persists to the config file тАФ without the
sandbox, recording a demo would rewrite your real `~/.config/clyde/config.toml`.

## Regenerating

Install VHS once:

```sh
brew install vhs
# or: see https://github.com/charmbracelet/vhs#installation
```

Make sure the `clyde` on your `$PATH` is fresh (`go install ./cmd/clyde`),
then regenerate from the **repo root**:

```sh
vhs demo/hero.tape
vhs demo/observability.tape
vhs demo/workspace.tape
vhs demo/customization.tape

# or all of them:
for tape in demo/*.tape; do vhs "$tape"; done
```

## Tuning

Each tape opens with a shared header:

```
Set Shell           "zsh"
Set FontFamily      "JetBrains Mono"
Set FontSize        16
Set Width           1400
Set Height          800
Set Padding         10
Set Theme           "TokyoNight"
```

If `JetBrains Mono` isn't installed, swap in any monospace font
(`Set FontFamily "Menlo"` works on stock macOS). At 1400├Ч800/16pt the
terminal is ~145 columns, which renders the 2-column medium layout; go
wider (тЙе160 cols) if you want to record the 3-column multi-col mode.

The tapes target ~10тАУ25s each so the GIFs stay small and start replaying
quickly on the README. If a clip balloons past ~3 MB, trim sleeps before
adding more demo steps.

## Updating after a TUI change

If you change a panel's rendering, layout, keybindings, or animation,
re-record the affected tape so the README stays honest. The contributor
checklist in [`CONTRIBUTING.md`](../CONTRIBUTING.md) flags this for
visual changes.
