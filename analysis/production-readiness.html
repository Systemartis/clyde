The two Critical findings

CRITICAL PR-01 · No --version flag

# bug_report.yml asks for it
- type: input
  attributes:
    label: Version
    description: Run `clyde --version`
# but main.go defines only:
flag.StringVar(&layoutFlag, "layout", "", "...")
flag.BoolVar(&demoFlag, "demo", false, "...")
flag.StringVar(&sourceFlag, "source", "claude", "...")

Verified: zero runtime/debug.ReadBuildInfo() in tree. Users today get "flag provided but not defined: -version" — a broken first impression.

CRITICAL PR-02 · No release infrastructure at all

  • No .goreleaser.yml (verified: find returns empty)
  • No release.yml workflow
  • No git tags (verified: git tag --list empty)
  • No multi-arch binaries · no Homebrew tap · no Scoop bucket
  • Only install path: go install from source — requires Go 1.26+ on user's machine
  • No SBOM · no cosign signatures · no SLSA provenance

Effort: M (~½ day for goreleaser config + release workflow + first v0.1.0 tag); blocks every other ship-it task.

Both findings sit in Phase 1 of the remediation plan. Until they ship, the rest of this work has nowhere to land.

High-severity findings

PR-04 go.mod:17-19 Three indirect deps on pseudo-versions (ultraviolet v0.0.0-20260416..., x/exp/golden v0.0.0-20251109..., terminfo v0.0.0-20220910...). Dependabot semver gating doesn't apply. S (coordinate w/ Charm + replace terminfo)

PR-06 repo root No pre-built binaries. ARM64 users (M1/M2 Macs, RPi, Graviton) must compile from source — slower, requires Go toolchain. S (folds into PR-02 goreleaser config)
PR-07 main.go:149-151
hookserver/server.go:148		 hs.URL() includes the auth token; full URL printed to stderr at startup. Users routinely paste startup output into bug reports — token leaks into shells, terminals, screenshots, screencasts. S (write URL to ~/.cache/clyde/hook-url mode 0600 instead)

PR-07 is operationally the most dangerous because it leaks at the moment users are most likely to share their terminal. The hookserver's per-process token is short-lived and loopback-only, but a leaked URL can be replayed by any local process while the session is alive.

Supply chain · current state

Direct deps minimal and reputable. The risk is in the indirect graph and the redirect hop.
