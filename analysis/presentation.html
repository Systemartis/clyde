What's installed under .claude/skills/

Each skill is a clyde-tuned rewrite of the upstream — references real files, real seams, the real depguard contract.

golang-pro

Use any time you touch a .go file.

  • Layer-first: domain → application → ports → adapters
  • Race detector mandatory
  • Bounded reads on every external io.Reader
  • Constant-time compare for tokens
  • Fuzz on JSONL/ps/git/OAuth/settings parsers

security-review

Use when touching secrets, hookserver, exec, HTTP, user-controlled paths.

  • Walks the SECURITY.md trust boundary
  • Shellout, file I/O, HTTP server, HTTP client, goroutine sub-checklists
  • Triage matrix to skip when not needed
  • Outputs analysis/SECURITY-FINDINGS.md entries

improve-codebase-architecture

Use when refactoring, adding adapters, or files pass ~500 LOC of branching.

  • Ousterhout vocabulary: depth, seam, leverage, locality
  • Deletion test, "two adapters = real seam"
  • Pre-named friction list (TUI keys.go, panel_viewer.go)
  • Positive examples (git.Source, claudesettings)

diagnose

Use before fixing any bug.

  • Build a deterministic loop < 2s
  • 3–5 falsifiable hypotheses, ranked
  • Catalog of test seams in clyde adapters
  • Regression test at the architectural seam closest to the bug
